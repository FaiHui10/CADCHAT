# CADCHAT 部署方式总结

本文档总结CADCHAT项目的两种部署方式及其特点。

## 部署方式概览

| 特性 | 局域网部署 (server/) | 阿里云部署 (aliserver/) |
|------|---------------------|-------------------------|
| 目录位置 | `server/` | `aliserver/` |
| 部署方式 | 直接运行Python脚本 | 直接运行或Docker部署 |
| Docker支持 | ❌ 不支持 | ✅ 支持 |
| 计算位置 | 本地 | 阿里云百炼平台 |
| 数据流向 | 完全本地化 | 发送到云端API |
| 硬件要求 | 高（需GPU） | 低（基础服务器） |
| 网络依赖 | 低 | 高（需互联网） |
| 数据安全 | 高（完全本地） | 中（云端处理） |
| 成本模型 | 一次性硬件投资 | API调用按量付费 |
| 维护难度 | 较高（需管理本地模型） | 较低（云端托管） |

## 局域网部署 (server/)

### 特点
- **本地化**：所有计算在本地完成
- **安全性**：数据不离开本地网络
- **硬件要求**：需要高性能GPU和大内存
- **部署方式**：直接运行Python脚本
- **无Docker**：不提供Docker部署选项

### 适用场景
- 对数据安全要求极高的环境
- 有足够硬件资源的企业
- 需要低延迟响应的场景
- 网络条件不佳的环境

## 阿里云部署 (aliserver/)

### 特点
- **云端计算**：使用阿里云百炼平台
- **轻量化**：对本地硬件要求低
- **部署灵活**：支持直接运行和Docker部署
- **Docker支持**：完整的容器化部署方案

### 适用场景
- 硬件资源有限的环境
- 需要快速部署的场景
- 云端服务优先的架构
- 小型团队或个人使用

## 选择建议

### 选择局域网部署的情况
- 数据安全是首要考虑
- 有高性能硬件资源
- 需要完全控制数据
- 网络连接不稳定

### 选择阿里云部署的情况
- 硬件资源有限
- 需要快速上线
- 云端原生架构
- 成本敏感型项目

## 部署流程

### 局域网部署流程
1. 安装Ollama和模型
2. 安装Python依赖
3. 启动服务端
4. 配置客户端

### 阿里云部署流程
1. 获取阿里云API密钥
2. 配置环境变量
3. 选择部署方式（直接运行或Docker）
4. 启动服务端
5. 配置客户端

## 维护考虑

### 局域网部署维护
- 需要维护本地AI模型
- 监控硬件资源使用
- 定期更新模型
- 管理本地数据

### 阿里云部署维护
- 监控API调用费用
- 管理云端服务配置
- 处理网络连接问题
- 云端安全配置

根据您的具体需求和环境条件，选择最适合的部署方式。
